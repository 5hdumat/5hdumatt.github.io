HTTP API 설계

API URI 고민 회원 정보 관리 API를 요구사항대로 만든다고 가정해보자. 요구사항 회원 목록 조회 회원 조회 회원 등록 회원 수정 회원 삭제 URL 설계가 끝난 후 경로가 나왔다. 회원 목록 조회 /read-member-list 회원 조회 /read-member-by-id 회원 등록 /create-member 회원 수정 /update-member 회원 삭제 /delete-member 뭔가 그럴듯하다. 실제 현업에서도 위와 같은 설계가 많이 이루어진다. 하지만 이것은 좋은 URI 설계일까? 우리는 위와 같이 설계된 URI가 좋은 설계인지에 대해서는 다시한번 고민해 봐야 한다....
