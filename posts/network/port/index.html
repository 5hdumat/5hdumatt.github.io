PORT ?#

PORT는 배가 도착하는 항구라는 뜻이다.

포트는 마치 항구와 같다. 패킷이 목적지 IP까지 안전하게 도착하도록 도와준다.
그게 어떤 애플리케이션이든 말이다.

예를 들면 아래와 같다.

한번에 둘 이상 연결해야 한다면?#

우리는 PC에서 한 작업만 하지 않는다.

ZOOM회의를 하면서 몰래 게임을 켜놓고 웹 브라우저로 쇼핑을 한다고 가정해보자.

가정대로라면 클라이언트가 여러개의 서버와 통신을 해야한다.
요청한 모든 응답 패킷들이 클라이언트의 각자의 출발지 IP로 뒤죽박죽 전송될 것이다.
(이건 클라이언트가 서버로 요청을 보낼때도 마찬가지다.)

게임 패킷인지, 화상통화 패킷인지, 웹 브라우저의 패킷인지 IP 프로토콜만 가지고는 구분할 수 없다.

사실 이전에 봤던 TCP/IP 패킷에 PORT 정보가 있다.#

사실 TCP/IP 패킷에 출발지와 목적지의 PORT 정보가 포함되어 있어 프로세스를 구분할 수 있었다.

IP는 목적지 서버를 찾는 것
PORT는 목적지 서버 안에서 동작하는 프로세스를 구분하는 것이라고 이해하면 된다.

IP는 아파트 PORT는 동,호수#

IP는 아파트 PORT는 동, 호수이다.

IP(아파트)뿐만 아니라 PORT(동, 호수)까지 열어서 연결하기 때문에
같은 IP(아파트) 내에서 프로세스를 구분하여 패킷(택배)을 전송 할 수 있다.

또한 PORT는 할당 범위가 정해져 있으므로 참고하자.

  • 0 ~ 65535 할당 가능
  • 0 ~ 1023: 잘 알려진 포트는 사용하지 않는 것이 좋다.
    • FTP - 20, 21
    • TELNET - 23
    • HTTP - 80
    • HTTPS - 443

  • PORT
