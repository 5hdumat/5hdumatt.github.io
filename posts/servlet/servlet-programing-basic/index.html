[서블릿 프로그래밍 기본] 프로젝트 준비

August 21, 2021 · 서민규 | 열혈강의 자바 웹 개발 워크북

개발환경: IntelliJ IDEA 2021.2 (Utimate Edition)

목표#

  • 서블릿 프로그래밍을 통해 웹 애플리케이션 만드는 방법을 숙지한다.
  • Servlet 인터페이스를 이해한다.

웹 프로젝트 준비#

책은 이클립스 예제만 있어 IntelliJ Dynamic Web Project 생성 방법을 따로 숙지해야 했다.

프로젝트 생성#

  • 1. [New Project] -> [SDK만 선택] -> [Next]

#

2. [프로젝트 명과 경로 지정] -> [Finish]

#

3. [생성된 프로젝트 오른쪽 클릭] -> [Add Framework Support…]

#

4. [Web Application (4.0) 체크] -> [Versions 선택] -> [OK]
web.xml은 필요에 따라 선택한다.

#

5. 웹 프로젝트 생성 완료
[Eclipse Dynamic Project]와 비슷한 폴더 구조가 생성되었다.

  • src
    자바 소스 파일을 두는 폴더이다.
    앞으로 이 폴더에 서블릿 클래스나 필터, 리스너 등 필요한 모든 자바 클래스 파일을 둘 것이다. .properties도 이 폴더에 둔다.
#
  • web
    Eclipse의 WebContent와 동일하다.
    HTML, CSS, JavaScript, JSP, 이미지 파일 등 웹 콘텐츠를 두는 폴더이다. 웹 애플리케이션 서버에 배치할 때 이 폴더의 내용물이 그대로 복사된다.
#
  • Web / Web-INF
    웹 애플리케이션의 설정과 관련된 파일을 두는 폴더이다.
    이 폴더에 있는 파일은 클라이언트에서 요철할 수 없다. 따라서 HTML이나 JavaScript, CSS 등 클라이언트에서 요청할 수 있는 파일을 이 폴더에 두어서는 안된다.
#
  • Web / Web-INF / web.xml
    웹 애플리케이션 배치 설명서(Deployment Descriptor) 파일이다. 영어 표현을 줄여서 DD 파일이라고도 부른다.
    서블릿이나 필터, 리스너, 매개변수, 기본 웹 페이지 등 웹 애플리케이션 컴포넌트들의 배치 정보를 이 파일에 작성한다. 서블릿 컨테이너는 클라이언트의 요청을 처리할 때 이 파일의 정보를 참고하여 서블릿 클래스를 찾거나 필터를 실행하는 등의 작업을 수행한다.
#
  • Web / Web-INF / lib
    자바 아카이브(Archive) 파일(.jar)을 두는 폴더이다. IntelliJ의 경우 필요하면 디렉토리를 추가 생성하면 된다.
    아카이브 파일은 클래스 파일(.class)과 프로퍼티 파일(.properties)들을 모아 놓은 보관소 파일이다. 이러한 이유로 Java ARchive의 합성어를 확장자명 .jar로 사용하고 있다.

#

  • 6. [Add Configuration] -> [설치된 톰캣으로 서버 설정]
    • 하단 [Fix] 클릭
    • [artifacts] 설정 -> [OK]]

7. 톰캣 실행 후 정상 확인

서블릿 클래스 생성 버튼을 찾을 수 없다면?#

  • [src] -> [New] -> [Servlet] -> ?

  • [project] 오른쪽 클릭 -> [Open Module Settings]

  • [Libraries] -> [From Maven…] 클릭

  • [javax.servlet] 검색 후 버전 선택

  • [OK] -> [모듈 선택]

  • 서블릿 클래스를 생성할 수 있게 되었다.
    우리는 책의 예제대로 lesson03 패키지 생성 후 HelloWorld 서블릿 클래스를 생성하도록 하자.

프로젝트 준비가 끝났으니 다음 포스트에서 서블릿을 직접 만들어보고 구동 원리를 이해해보도록 하자.
